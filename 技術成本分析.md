### 五、成本優化建議

#### 5.1 Prompt Caching（省 50-70% 成本）

```javascript
// 使用 Claude 的 Prompt Caching
const systemPrompt = `
你是專業房地產 AI 助理...
（長達 2000 tokens 的系統提示詞）
`;

// 標記為可快取
anthropic.messages.create({
  model: "claude-haiku-4",
  system: [
    {
      type: "text",
      text: systemPrompt,
      cache_control: { type: "ephemeral" }  // ← 啟用快取
    }
  ],
  messages: [...]
});

// 效果：
// - 第 1 次：2000 tokens × $0.25 = $0.50
// - 第 2-100 次：2000 tokens × $0.03 (快取價) = $0.06
// 省下 88% 成本！
```

#### 5.2 批量處理（減少 API 呼叫次數）

```javascript
// ❌ 錯誤做法：每個物件單獨呼叫
for (const listing of listings) {
  await claude.judge(listing);  // 15 次 API 呼叫
}

// ✅ 正確做法：批量判斷
await claude.judgeBatch(listings);  // 1 次 API 呼叫
```

**效果**：
- API 呼叫次數：15 次 → 1 次
- 總 Token 消耗：35,250 → 25,000（省 30%）

#### 5.3 智能過濾（減少無效判斷）

```javascript
// 明顯仲介物件直接排除（不呼叫 API）
if (listing.contact.includes('0800') || 
    listing.title.includes('房屋網')) {
  return { score: 0, reason: '明顯仲介' };
}

// 只對「可能是屋主」的物件呼叫 LLM
```

**效果**：
- 15 個物件 → 實際呼叫 8 個（省 47%）
- 月費：NT$ 120 → NT$ 65

## 📞 常見問題

### Q1: 如果客戶用量暴增怎麼辦？
**A**: 系統會自動發送警報，建議：
1. 降低監控頻率（2h → 4h）
2. 減少監控區域
3. 切換至 Gemini Flash（成本降 70%）

### Q2: API Key 會不會被盜用？
**A**: 
- API Key 僅儲存於客戶電腦（加密）
- 設定每月預算上限（如 $10）
- 異常呼叫自動停止

### Q3: 為什麼不用免費的 LLM？
**A**: 
- 免費模型（Llama、Mistral）準確度不足（<70%）
- 推薦物件錯誤率高，浪費客戶時間
- 商用 LLM 成本極低（每月 <$5）

### Q4: 可以用客戶自己訓練的模型嗎？
**A**: 可以，但需要：
- 模型支援 API 呼叫
- 準確度 >80%（需實測）
- 回應時間 <2 秒
