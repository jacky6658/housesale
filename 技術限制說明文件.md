## ⚠️ 半自動化平台（需人工協助）

### 4. Facebook 社團

**技術挑戰：**
1. **登入需求**
   - Facebook 要求帳號登入才能瀏覽社團
   - 反爬蟲機制非常嚴格
   - 頻繁自動化操作容易被封鎖帳號

2. **動態網頁結構**
   - React 框架，頁面結構動態變化
   - 元素選擇器不穩定
   - 需額外處理 JavaScript 渲染

3. **帳號風險**
   - 自動化操作違反 Facebook 服務條款
   - 被偵測後可能永久封鎖帳號
   - 需使用「人類化」操作模擬（隨機延遲、滑鼠軌跡）

**可行方案：**
```
步驟 1：客戶手動登入 Facebook（一次性）
   ↓
步驟 2：AI 代理接管，模擬人類行為監控
   ↓
步驟 3：每日監控時間限制（2-3 次，避免觸發風控）
   ↓
步驟 4：發現高分物件 → 推送給客戶
```

**風險評估：**
- 🟡 **中風險**：帳號可能被限制功能（暫時性）
- 🔴 **高風險**：帳號被永久封鎖（機率 10-20%）

**建議策略：**
1. **選項 A（保守）**：僅使用 AI 協助分析，客戶手動瀏覽
2. **選項 B（積極）**：AI 半自動監控，低頻操作（每日 2 次）
3. **選項 C（放棄）**：不納入 Facebook，專注 591/PTT

**我們的建議：選項 B（半自動）**
- 登入由客戶負責（降低風險）
- AI 僅執行低頻監控（每日 2-3 次）
- 發現可疑風控立即停止

## ❌ 無法自動化平台

### 5. LINE 社團

**技術限制：**
1. **封閉生態系統**
   - LINE 是完全封閉的通訊平台
   - 無法透過網頁瀏覽器存取社團內容
   - 必須使用 LINE App

2. **缺乏公開 API**
   - LINE 官方未提供社團內容存取 API
   - LINE Bot API 僅能接收訊息，無法主動爬取
   - 第三方工具均違反服務條款

3. **法律風險**
   - 破解 LINE 協議屬於違法行為
   - 可能觸犯《電腦處理個人資料保護法》

**替代方案：**
```
方案 1：手動管理
- 客戶自行瀏覽 LINE 社團
- 發現物件後轉傳給 AI 代理分析

方案 2：LINE Notify 整合（未來規劃）
- 客戶手動複製物件連結
- 透過 LINE Notify 傳送給 AI 系統
- AI 自動分析並回覆評分

方案 3：LINE Bot API 開發（另行報價）
- 開發專屬 LINE Bot
- 客戶手動轉傳物件訊息給 Bot
- Bot 自動分析並回覆
- **額外開發成本：50K-80K**
```

**我們的建議：方案 1（手動管理）**
- LINE 社團物件量通常不大（每日 5-10 筆）
- 手動管理成本低於開發成本
- 未來若需求增加再評估 Bot 開發

## 📞 客戶常見問題

### Q1：為什麼 Facebook 不能完全自動化？

### Q2：如果 Facebook 帳號被封鎖怎麼辦？
**A：** 
1. 我們會提供「人類化」操作降低風險
2. 建議使用備用帳號（非主帳號）
3. 如被封鎖，可切換回 591/PTT 專注模式
4. 帳號風險由客戶承擔（已在合約中說明）

## 📋 附錄：技術名詞解釋

**反爬蟲機制（Anti-Scraping）**  
網站用來阻止自動化程式存取資料的技術，如驗證碼、IP 封鎖、請求頻率限制等。

**人類化操作（Human-like Behavior）**  
模擬真人使用習慣的自動化操作，包含隨機延遲、滑鼠軌跡、捲動行為等，降低被偵測機率。

**API（Application Programming Interface）**  
應用程式介面，官方提供的資料存取方式，合法且穩定。

**服務條款（Terms of Service, ToS）**  
平台規定使用者行為的法律文件，違反可能導致帳號被封鎖。
